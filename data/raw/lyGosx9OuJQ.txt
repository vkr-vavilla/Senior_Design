For today's software engineering interview, please explain webhooks. A webhook provides a way for other applications to let your application know when an event happens. But let me explain this through a simple example. Let's say I'm using Stripe to process payments, and I have some application which does other application logic. Whenever I want to know if there's a new payment, I have to make an API call to Stripe to get the payments, and Stripe can return me a list of all the new payments. Making constant API requests to look for new data there does seem like a pain in this case. Exactly. It's slow because I might not make an API call right when a payment happens, and it wastes a lot of resources because I'm constantly making API calls to check for new payments whether or whether not somebody actually made one. So, here's where webhooks come in. I just send Stripe a URL of where my webhook is on the server side, and from then on out, anytime any event happens, in this case a payment, Stripe can send a post request with the information that I need. Okay, and then based on that notification, you can do something in real time. Exactly. Now, I'm not wasting resources making API requests to look for events that may or may not have happened, and as soon as an event happens, I'm able to do something like send my users an email thanking them for buying.
